## Google Cloud Global Infrastructure

### Google Cloud operates a global private fiber network connecting:

- Regions
- Zones
- Edge Points of Presence (PoPs)

### Benefits include:

- Low latency
- High availability
- Global scalability
- Secure software-defined networking

### ACE Recognition Pattern

| Global applications → Google Global Network

## Cloud Service Models
| Model |	Responsibility |
| ----- | -------------- |
| IaaS	| Customer manages VMs and OS |
| PaaS	| Google manages platform |
| SaaS	| Google manages entire application |

### Examples:

- IaaS: Compute Engine
- PaaS: GKE, Cloud Run
- SaaS: Google Workspace

---

## Infrastructure Analogy

Google compares cloud infrastructure to a city.

| City |	Google Cloud |
| ---- | ------------- |
| Roads	| Networking |
| Power	| Compute |
| Buildings	| Applications |
| Citizens	| Users |
| Utilities	| Infrastructure |

This is a memorable analogy for interviews and the ACE exam.

## Cloud SQL vs Self-managed MySQL

### Self-managed MySQL:

- Create VM
- Install MySQL
- Patch OS
- Backups
- Security

### Cloud SQL:

- Managed backups
- Managed patching
- Automatic maintenance
- Less operational overhead

### ACE Recognition Pattern

Need managed relational database → Cloud SQL
